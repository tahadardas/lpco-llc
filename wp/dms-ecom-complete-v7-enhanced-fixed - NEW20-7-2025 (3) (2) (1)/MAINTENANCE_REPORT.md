## 🔧 المشاكل المحددة والحلول المطبقة

### ❌ المشكلة 1: فقدان 1060 منتج (15.8% من البيانات)
**السبب:** نقص في آليات البحث عن المنتجات
- `wc_get_product_id_by_sku()` لا تعمل دائماً
- الـ `meta_query` الوحيد غير كافي

**✅ الحل المطبق:**
```php
function dms_find_product_by_sku($sku) {
    // 1. البحث المباشر
    $product_id = wc_get_product_id_by_sku($sku);
    if ($product_id) return $product_id;
    
    // 2. البحث عبر Meta Query
    $products_found = get_posts([...]);
    if (!empty($products_found)) return $products_found[0]->ID;
    
    // 3. البحث عبر Post Slug
    $post = get_page_by_path($sku, OBJECT, 'product');
    if ($post) return $post->ID;
    
    return null;
}
```

---

### ❌ المشكلة 2: معلومات الطرد والعلبة لا تُحفظ
**السبب:** استخدام `update_post_meta()` التي تُرجع false عند عدم التغيير

**✅ الحل المطبق:**
```php
// استخدام نمط القذف والإضافة
delete_post_meta($product_id, '_dms_prices');
$result = add_post_meta($product_id, '_dms_prices', $meta);

// إذا فشل، يمكن المحاولة مع update
if (!$result) {
    update_post_meta($product_id, '_dms_prices', $meta);
}
```

### ❌ المشكلة 3: توقف الاستيراد (503 Service Unavailable)
**السبب:** استهلاك موارد مفرط من:
- الكتابة إلى الملف 46,928 مرة (log per row)
- تحميل جميع المنتجات للبحث عن SKUs مشابهة

**✅ الحلول المطبقة:**
1. تقليل التسجيل: كل 500 صف فقط
2. إزالة البحث عن SKUs مشابهة
3. زيادة حد الوقت إلى 10 دقائق
4. `ignore_user_abort(true)` للحفاظ على العملية

### ❌ المشكلة 4: القيم الافتراضية الفارغة
**السبب:** عدم حفظ القيم الافتراضية أثناء الاستيراد

**✅ الحل المطبق:**

```php
$meta[$cat]['box_unit_name'] = !empty($box_unit_name) ? $box_unit_name : 'علبة';
$meta[$cat]['package_unit_name'] = !empty($package_unit_name) ? $package_unit_name : 'طرد';
```

## 📝 تنسيق ملفات CSV

### 1️⃣ ملف الأسعار (9 أعمدة)
```
اسم المنتج | SKU | التصنيف | سعر القطعة ل.س | سعر القطعة $ | عدد القطع في العلبة | اسم وحدة العلبة | عدد القطع في الطرد | اسم وحدة الطرد
```

```
SKU | اسم المنتج | الأولوية | اتجاه العرض
```

**مثال:**
```
SKU-001 | دفتر A5 | 1 | متصاعد
```

## 📞 استكشاف الأخطاء

### مشكلة: منتجات لا تُعثر عليها
**الحل:**
1. تحقق من SKU (هل به مسافات أو أحرف خاصة؟)
2. تحقق من الاسم في قاعدة البيانات
3. جرب البحث اليدوي في WooCommerce

### مشكلة: الاستيراد يتوقف عند صف معين
**الحل:**
1. تحقق من صيغة البيانات (ترميز UTF-8)
2. جرب ملف أصغر
3. راجع السجل للرسائل التفصيلية

### مشكلة: البيانات لا تُحفظ
**الحل:**
1. تحقق من صلاحيات المجلد `/wp-content/`
2. تحقق من قاعدة البيانات
3. جرب تحديث يدوي لمنتج واحد
